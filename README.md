# MCSR 全类型种子自动输入工具（增强版）
专为 MCSR Ranked 速通练习设计，支持多类型开局种子自动获取与一键输入，替代手动复制粘贴，大幅提升速通练习效率。新增了下界堡垒/要塞变种筛选、Elo 匹配、比赛 ID 查询并导入相同变种、高级设置折叠面板等功能，帮助玩家更精准地练习特定种子。

## 功能特性
1. 自动调用 API 获取 6 种主流开局类型种子（宝藏/废门/沙漠神殿/村庄/沉船/随机)
2. 支持多类型多选，后台预加载下一批种子，保证操作连贯性
3. 纯键盘逐字符模拟输入，避免剪贴板粘贴的输入错误
4. 严格使用 MCSR Ranked 排位赛 sub12 种子库（默认 completion=720000ms）
5. 新增下界堡垒类型筛选（桥/藏宝室/居住区/棚）和下界要塞变种筛选（5 种生物群系）
6. 支持主世界、下界、末地三大类变种精细筛选，可自由组合
7. Elo 权重系统：可根据玩家分段（0-599/600-1199/1200+）自动调整开局类型的出现概率，也可手动自定义权重
8. 百宝箱功能: 通过比赛 ID 查询历史种子的类型、变种信息，并一键导入到当前设置中，快速复现特定种子
9.  API 地址自定义：支持修改 API 端点，清空输入框自动恢复默认地址
10. 所有设置（勾选、权重、热键等）自动保存至 config.json，下次启动自动加载
11. 日志窗口独立于右侧，可滚动查看详细操作记录，支持清空和导出日志

## 环境准备
### 1. 基础要求
(1) 已安装 Python 3.8 及以上版本（推荐 3.10+）
(2) 确保网络可访问 MCSR Ranked API 地址（默认 http://43.143.231.104:8001）

### 2. 安装依赖库
打开命令行执行以下命令：
```
pip install requests pynput pyinstaller
```

#### 依赖库说明：
(1) requests：发送网络请求，获取 API 种子数据
(2) pynput：监听键盘按键、模拟键盘输入
(3) pyinstaller：将 Python 脚本打包为可执行文件

## 使用方法
### 方式一：使用打包后的可执行文件（推荐）
1. 将程序打包为独立 exe：
   - 将封面图片（如 RANKED.png）与脚本放在同一目录
   - 修改脚本中的图片加载代码，使用 resource_path 函数（示例见下文）
   - 执行打包命令（以脚本名为 Ranked.py 为例）：
     ```
     pyinstaller -F -w --add-data "RANKED.png;." Ranked.py
     ```
     如需设置 exe 图标，可加 -i icon.ico
   - 打包完成后，在 dist 文件夹中找到 Ranked.exe
2. 双击运行 Ranked.exe
3. 在 GUI 中：
   (1) 勾选开局类型（可多选，全不选则从全部类型随机）
   (2) 若启用 Elo 权重，则开局类型复选框自动禁用，权重设置接管类型选择
   - 在“高级设置”中可自由勾选下界堡垒类型、各类变种，以及设置完成时间上限
   - 点击“百宝箱”可输入比赛 ID 查询历史种子信息，点击“导入到基本界面”即可将类型和变种填入主界面
4. 设置启动和退出热键（点击按钮后按下任意键）
5. 切换到 Minecraft 游戏主界面（确保窗口为活动状态）
6. 按开始热键自动完成种子输入，按退出热键安全关闭程序

### 方式二：直接运行 Python 脚本
1. 将脚本保存为 ranked10.py
2. 双击运行，或命令行进入脚本所在目录，执行：
   ```
   python Ranked.py
   ```
3. 后续操作同上。

## 配置文件说明
程序启动时会自动读取同目录下的 config.json，若不存在则使用默认设置。修改任意设置（勾选、权重、热键等）后自动保存。配置文件采用 JSON 格式，可手动编辑，但建议通过 GUI 操作。

## API 相关说明
核心调用 API：工具调用 MCSR Ranked v2 版 API，根据筛选条件构造 URL：
```
http://43.143.231.104:8001/api/v2/seed?overworld={类型}&nether={类型}&variations={变种列表}&completion={毫秒}
```

- overworld ：主世界类型标识（buried_treasure / ruined_portal / desert_temple / village / shipwreck）
- nether    ：下界堡垒类型标识（bridge / treasure / housing / stable）
- variations：变种条件，多个用逗号分隔（如 biome:structure:plains,bastion:triple:2）
- completion：完成时间上限（毫秒），限定为排位模式种子，默认 720000（12 分钟）

完整 API 文档请访问：

## 注意事项
1. 运行程序前，必须将 Minecraft 窗口置于最前，且确保为当前活动窗口。
2. 种子输入过程中有微小字符延迟（0.01 秒/字符），为保证输入准确，请勿在执行过程中手动操作键盘/鼠标。
3. 若程序无响应，优先按退出热键（默认 F6）。
4. 打包可执行文件时，确保文件名无中文/特殊字符，建议以管理员身份运行命令行。
5. 工具仅可用于 MCSR Ranked 速通练习，请勿用于违规游戏场景。
6. 预加载失败时请检查筛选条件是否过严（如变种组合过于稀有），可适当放宽条件。

## 常见问题排查
### Q1：按下启动热键无反应？
A1：① 确认 Minecraft 窗口为当前活动窗口；② 检查程序是否正常运行（日志无报错）；③ 关闭其他占用键盘的软件（如输入法、按键精灵等）。

### Q2：种子输入出现缺字/错字？
A2：① 确保游戏内输入框为英文输入法状态；② 不要在程序执行时手动操作键盘，等待输入流程完成。

### Q3：打包 exe 时提示报错？
A3：① 升级 pyinstaller 到最新版本：`pip install --upgrade pyinstaller`；② 确保文件路径无中文/特殊字符；③ 以管理员身份运行命令行重新打包；④ 若缺少 pynput 模块，可添加 --hidden-import pynput.keyboard._win32。

### Q4：API 获取种子失败（404 或 500）？
A4：① 检查网络连接，能否正常访问 API 地址；② 检查 API 地址是否正确；③ 查看当前筛选条件（变种、下界类型、完成时间）是否过于严格，导致无符合条件的种子；④ 若出现“can only concatenate list...”错误，为服务器端 bug，请向 API 维护者反馈。

### Q5：百宝箱导入后预加载频繁失败？
A5：导入的种子变种可能非常稀有，请在“高级设置”中适当减少变种勾选或清空“其他变种”文本框，放宽筛选条件。
